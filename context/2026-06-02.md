# BabyFlow — Session Context 2026-06-02

## What this app is
BabyFlow is a baby event tracking web app. Parents log sleep, wake-up, feeding, and diaper events and see them visualised on a per-day timeline with charts for trends.

## Tech stack
- **Framework**: Next.js 16 (App Router, React 19)
- **Language**: TypeScript (strict)
- **Styling**: Tailwind CSS v4
- **DB / Auth**: Supabase (Postgres + magic-link auth via `@supabase/ssr`)
- **ORM**: Drizzle ORM + `postgres` driver
- **Charts**: Recharts (client-only via `dynamic()` wrappers) + custom SVG timeline
- **i18n**: next-intl v4 — Spanish default (`/es`), English (`/en`)
- **Testing**: Vitest + React Testing Library
- **Deployment**: Vercel (not yet deployed, local only as of this session)
- **Repo**: `https://github.com/brais1912/baby-flow` (private, personal account `brais1912`)

## Database schema (Drizzle)
Table: `events`
| Column | Type | Notes |
|---|---|---|
| id | uuid PK | defaultRandom() |
| user_id | uuid | Supabase auth user ID |
| type | enum | `sleep`, `wake_up`, `feeding`, `diaper` |
| occurred_at | timestamptz | When the event happened |
| notes | text | Optional |
| sleep_method | enum | `nursing`, `bottle`, `pacifier`, `held`, `rocking`, `self`, `other` |
| sleep_condition | enum | `sleep_sack`, `pajamas`, `swaddle`, `other` |
| sleep_room_temperature | real | °C, default 25 in form |
| feeding_type | enum | `breast_left`, `breast_right`, `bottle`, `formula`, `solid` |
| feeding_amount_ml | real | Only for bottle/formula |
| feeding_duration_minutes | real | |
| diaper_type | enum | `pee`, `poop`, `both` |
| created_at | timestamptz | defaultNow() |
| updated_at | timestamptz | defaultNow() |

Migrations live in `supabase/migrations/`. Three migrations applied so far (0000, 0001, 0002).

## Key design decisions made this session

### Event model
- `sleep` and `wake_up` are **separate events** — sleep = fell asleep, wake_up = woke up
- Sleep duration is calculated by pairing each `sleep` with the next `wake_up` chronologically
- "Both breasts" feeding creates **two events** (breast_left + breast_right) at the same timestamp

### Dashboard layout (top to bottom)
1. Title + subtitle ("Resumen de hoy · navega por días abajo")
2. Stat cards: 😴 Sleeping events / 🍼 Feedings / 👶 Diapers (today's counts)
3. **DayView** — the main interactive card:
   - Day navigator (‹ Hoy ›)
   - Filter buttons: Todo / 😴 Sueño / 🍼 Tomas / 👶 Pañales
     - "Sueño" filter shows BOTH `sleep` and `wake_up` events together
   - Event table: Time | Type badge | Details (wake_up row shows sleep→wake time + duration)
   - Custom SVG timeline (see below)
4. Sleep duration bar chart (Recharts)
5. Feeding chart — breast sessions stacked bar + bottle/formula line chart
6. Diaper chart — pie (by type) + bar (per day)

### Timeline chart (custom SVG, no Recharts)
- **Multi-lane**: each event type gets its own Y row (😴 Sueño / 🍼 Tomas / 👶 Pañales)
- Empty lanes hidden; chart height adjusts dynamically
- Sleep sessions rendered as **horizontal bars** (purple bar from sleep time to wake-up time)
  - Purple dot = fell asleep, orange dot = woke up
- Other events rendered as dots
- Clicking a bar/dot opens a bottom-sheet detail popup
- Session pairing looks ahead up to 24h (handles overnight sleep crossing midnight)
- `events` prop = full 7-day set (for session pairing); `visibleEvents` prop = filtered subset (controls what's shown)

### i18n
- Server Components use `getTranslations()` / `getLocale()` from `next-intl/server`
- Client Components use `useTranslations()` / `useLocale()` from `next-intl`
- **IMPORTANT**: Never use `useTranslations`/`useLocale` in Server Components — this caused a React "suspended thenable" crash that was fixed by switching to the server variants

### Auth
- Magic link via Supabase. The `?code=` param lands on `/[locale]/login` and is exchanged client-side via `supabase.auth.exchangeCodeForSession(code)`
- Supabase dashboard → Authentication → URL Configuration:
  - Site URL: `http://localhost:3000`
  - Redirect URLs: `http://localhost:3000/**`

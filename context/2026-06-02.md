# BabyFlow — Session Context (updated 2026-06-03)

## What this app is
BabyFlow is a baby event tracking web app. Parents log sleep, wake-up, feeding, and diaper events and see them visualised on a per-day timeline with charts for trends.

## Tech stack
- **Framework**: Next.js 16 (App Router, React 19)
- **Language**: TypeScript (strict)
- **Styling**: Tailwind CSS v4
- **DB / Auth**: Supabase (Postgres + email/password auth via `@supabase/ssr`)
- **ORM**: Drizzle ORM + `postgres` driver
- **Charts**: Recharts (client-only via `dynamic()` wrappers) + custom SVG timeline
- **i18n**: next-intl v4 — Spanish default (`/es`), English (`/en`)
- **Testing**: Vitest + React Testing Library
- **Deployment**: Vercel (not yet deployed, local only as of this session)
- **Repo**: `https://github.com/brais1912/baby-flow` (private, personal account `brais1912`)

## Key design decisions

### Event model
- `sleep` and `wake_up` are **separate events** — sleep = fell asleep, wake_up = woke up
- Sleep duration is calculated by pairing each `sleep` with the next `wake_up` chronologically
- "Ambos pechos" (both breasts) is stored as **one event** with `feedingType = "both_breasts"` (migration 0004 added this enum value). Old data may have two rows (breast_left + breast_right at same timestamp) — `deduplicateBothBreasts()` in `format.ts` collapses those for display.

### Dashboard layout (top to bottom)
1. Title + subtitle
2. Stat cards: 😴 / 🍼 / 👶 counts — **synced to the selected day** via `DashboardClient`
3. **DayView** — day navigator (‹ Hoy ›) + filter pills + event cards + SVG timeline
4. Sleep duration bar chart
5. Feeding chart (breast sessions per day + bottle/formula line)
6. Diaper chart (stacked bar: pis / caca / ambos)

### Recharts chart interaction (tap → popover)
- **Rectangle suppression**: `style={{ pointerEvents: "none" }}` on every `ResponsiveContainer` — Recharts SVG never receives touch events, so it never enters active state and draws no highlight rectangle.
- **Tap detection**: wrapper `<div onPointerUp>` reads `clientX` + `clientY`, maps X to bar index via `tapBarIndex()` using each chart's margin values.
- **Popover**: small card positioned at `clientY` (above tap if near bottom of screen, below otherwise). NOT a bottom sheet.
- Never use Recharts `onClick` on `BarChart`/`LineChart` — it goes through Recharts' touch state machine and triggers the rectangle.

### Timeline chart (custom SVG — src/components/dashboard/TimelineChart.tsx)
- Multi-lane SVG: each type gets its own Y row
- Sleep sessions = horizontal bars (purple dot = sleep, orange dot = wake_up)
- Other events = dots; click opens a detail popover (bottom-sheet style, separate from charts popover)
- `events` prop = full 7-day set (for session pairing); `visibleEvents` = filtered subset

### Time picker (EventForm)
- Custom component: 3 day buttons (Hoy / Ayer / day-before) + hour select + minute select (5-min steps)
- Replaces `datetime-local` input for better mobile UX
- `buildDate(dayOffset, hour, minute)` constructs the final `Date`

### Auth
- Email + password via `supabase.auth.signInWithPassword()` and `supabase.auth.signUp()`
- Password reset: login page → `resetPasswordForEmail` with `redirectTo = /{locale}/auth/reset-callback` → that route exchanges the code → redirects to `/{locale}/reset-password` → `supabase.auth.updateUser({ password })`
- Middleware: `isResetPassword` is a separate flag — authenticated users are NOT redirected away from `/reset-password`
- Supabase dashboard → Authentication → URL Configuration:
  - Site URL: `http://localhost:3000`
  - Redirect URLs: `http://localhost:3000/**`

### i18n
- Server Components: `getTranslations()` / `getLocale()` from `next-intl/server`
- Client Components: `useTranslations()` / `useLocale()` from `next-intl`
- **Never** use `useTranslations`/`useLocale` in Server Components — causes React "suspended thenable" crash
- Event type pills in DayView use `tEventTypes(type)` translations, NOT the hardcoded `eventTypeLabel()` utility

### Timezone handling
- Server fetches events from `startOfDay(subDays(now, 8))` to `startOfDay(addDays(now, 1))` in UTC — wide window covers Spain (UTC+2) correctly
- Charts group by local date via `toLocaleDateString("es-ES" | "en-US")`
